Don't worry. It's not complicated even if you don't know anything about optimization.
Input:
First of all, the input is an expression for the objective function, and an inequality for the constraints.
You can write expressions quite flexibly. The following are an example of some valid expressions:
x+y OR .5x - -6.y OR x_1 + -10 * x_2
Inequalities must contain bigger than ">" or less than "<" symbol. Adding "=" after them ("<=", ">=") is optional.
There is a button called "Clear all", which you should not press if you don't want to lose your input.
You can add or remove constraints using the two buttons below. Minimum is 1 constraint, and maximum is 15.
Options:
There are also a few options. You can choose if you want to maximize or minimize (quite useless IMHO).
Also, you can choose to input the constraints in one box for your convenience (they should be seperated by new lines).
Solution append is a helpful option.
If you're solving an LP, you can either choose to show all the pivots, or just show the final results.
If you're solving an IP, You can choose to see every single step in every LP, or just show the initial LPs and their results.
Of course, if you would like to solve the LP assuming the variables are integers, you can set it to integer programming mode.
Finally, you can set the precision of the shown numbers. Allowed precisions are from 0 to 7, else the precision will default to 2.
Solving:
In LP mode, "Start" initializes the LP, "Pivot" makes a single pivot, and "Solve LP" solves the goddamn LP (do all possible pivots).
When not in solution-append mode, you can view previous (or already done) pivots by pressing the arrows beside "Solution".
In IP mode, "Solve IP" solves the LP assuming integers only. You can view the pivots of all LPs by choosing solution-append mode in options.
"Final Result" simply shows the final result, but only the final result and nothing else.
Finally, this sophisticated button called "IP Tree" prints the branch-and-bound IP tree, with the process and the result of each LP.

Hint: if the solution looks weird, or your constraints are too long, try dragging the box to properly view your stuff.
Example: maximize 5 x1 + 8 x2, subject to x1 + x2 <= 6 and 5 x1 + 9 x2 <= 45.
